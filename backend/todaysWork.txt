Request → Routes → Controller → Service → Prisma ORM → Database
                    ↓

                    ↓

- **Route Layer**: Defines endpoints and applies middleware

- **Middleware Layer**: Cross-cutting concerns (authentication, authorization)
- **Validation Layer**: Input schema validation using Joi

## **5. Security & Authentication Implementation**

### **5.1 JWT Authentication**

**Token Structure:**
- **Algorithm**: HS256 (HMAC-SHA256)
- **Secret**: Stored in `process.env.JWT_SECRET`
- **Payload**: `{ id: number, role: string }`
- **Expiration**: 1 day (`"1d"`)

**Token Location:** 
- Sent in request header: `Authorization: Bearer <token>`

**Token Verification:**javascript
// authMiddleware logic:
1. Extract token from "Authorization" header
2. Split by space and take second part
3. Verify signature using JWT_SECRET
4. Attach decoded payload to req.user
5. Pass to next middleware/controller

**Error Handling:**
- 401 if token missing: "Token required...!"
- 401 if token invalid/expired: "Invalid token"

### **5.2 Password Security**

**Hashing Algorithm:** bcryptjs
- **Salt Rounds**: 10
- **Hash Function**: bcrypt (adaptive, resistant to brute-force)
- **Cost Factor**: Automatic with 10 rounds (∼10ms per hash on modern hardware)

**Password Validation:**
Pattern: /^(?=.*[A-Z])(?=.*[\W_]).{8,16}$/
- Must contain at least one uppercase letter (A-Z)
- Must contain at least one special character (non-word or underscore)
- Must be 8-16 characters long

**Examples:**
- ✅ Valid: `MyPass!123`, `SecureP@ss99`, `Test#1234`
- ❌ Invalid: `Password123` (no special char), `test@` (no uppercase), `short!` (< 8 chars)

**Password Change Flow:**
1. User provides old and new password
2. Old password verified with bcrypt.compare()
3. New password hashed with bcryptjs (10 rounds)
4. Update in database

### **5.3 Role-Based Access Control (RBAC)**

**Role Types:**
Enum Role {
  ADMIN         // Full system access
  USER          // Can rate stores, view store listings
  STORE_OWNER   // Can view own store dashboard, receive ratings
}

**Middleware Implementation:**javascript
// roleMiddleware accepts one or more allowed roles
router.get("/admin/users", 
  authMiddleware,                    // Step 1: Verify JWT
  roleMiddleware("ADMIN"),           // Step 2: Check role
  AdminController.getUsers           // Step 3: Execute endpoint
);

**Route-Level Access Control:**

| Route | Required Role | Middleware Chain ||
| `/auth/*` | None | None |
| `/admin/*` | ADMIN | authMiddleware → roleMiddleware("ADMIN") |
| `/ratings/*` | USER (any authenticated) | authMiddleware |
| `/stores/*` | USER (any authenticated) | authMiddleware |
| `/owner/dashboard` | STORE_OWNER | authMiddleware → roleMiddleware("STORE_OWNER") |

**Access Denied Response:**json
{
  "success": false,
  "message": "Access denied...!"
}
// Status: 403 Forbidden

### **5.6 Database Security**

**Connection:**
- Stored in `process.env.DATABASE_URL`
- Credentials kept in environment variables (not in code)
- Prisma manages connection pooling

**Foreign Key Constraints:**
- Relationships enforced at database level
- Prevents orphaned records
- Cascade deletes configured in schema

**Unique Constraints:**
- Email uniqueness: `User.email`, `Store.email`
- Composite unique: `Rating [userId, storeId]`
- Prevents business logic violations at DB level

### **5.7 Error Handling & Information Disclosure**

**Sensitive Data Exclusion:**

## 1. High-Level Overview

This is a React single-page application built with Vite that serves as an owner-facing dashboard for store owners. Its core purpose is to authenticate a store owner, authorize only the `STORE_OWNER` role, and present an owner dashboard with store data and ratings. The app is frontend-only in this workspace; it integrates with a backend API hosted at `http://localhost:5000`.

## 2. Folder Structure Breakdown

- src
  - main.jsx

Architectural pattern:
- Feature-based organization with Redux Toolkit slices per domain (`auth`, `owner`).
- React Router for page routing.
- Centralized API client for backend communication.
- Local storage persistence for auth token and user data.

## 3. Core Functionalities

- Authentication
  - Login with email/password.
  - Stores JWT token and user payload in `localStorage`.

- Authorization / Route protection
  - Only authenticated users with `user.role === "STORE_OWNER"` can access dashboard and password change pages.

- Dashboard data retrieval
  - Fetches owner dashboard data from `/owner/dashboard`.
  - Renders stores and rating details.

- Password management
  - Sends password change requests to `/auth/change-password`.

- Client-side session management
  - Logout clears local storage and Redux auth state.
  - Redirects to `/dashboard` after successful login.
  - Redirects unauthorized or unauthenticated users back to login.

- Store/rating exploration
  - Dashboard displays a list of stores and toggles details for each store.
  - Shows rating entries per store.

## 6. API Endpoints / Data Flow

Frontend interacts with backend through a single Axios instance:
- axios.js
  - `baseURL: http://localhost:5000`
  - attaches `Authorization` header from `localStorage.token`

Implemented endpoints:
- `POST /auth/login`
  - Used by `loginUser`.
  - Expects login credentials.
  - Returns `data` containing `{ token, user }`.
  - Frontend enforces `STORE_OWNER` role before allowing login success.

- `PUT /auth/change-password`
  - Used by `changePassword`.
  - Sends `{ oldPassword, newPassword }`.

- `GET /owner/dashboard`
  - Used by `getOwnerDashboard`.
  - Returns owner dashboard data with store ratings.

Data flow:
1. User submits login form.
2. `loginUser` thunk posts to `/auth/login`.
3. On success, auth slice stores token, user, and sets `isAuthenticated`.
4. ProtectedRoute validates auth + role and allows access.
5. Dashboard page dispatches `getOwnerDashboard` thunk.
6. Dashboard slice stores returned dashboard data.
7. Dashboard.jsx renders stores + ratings from Redux state.
8. ChangePassword.jsx dispatches password update and reacts to success.
